Topics Covered:
- Learned about TextField vs CharField in Django
  - TextField is for long/unbounded text (maps to TEXT in Postgres)
  - CharField requires max_length (maps to VARCHAR)
  - No real performance difference in PostgreSQL

- Learned about the __str__ method in Django models
  - Controls how a model instance displays as a string
  - Used in admin, shell, dropdowns, and debugging
  - Must use double underscores (__str__), not single (_str_)

- Renamed the games model to Game
  - Followed Django convention: singular PascalCase
  - Added a moves TextField to the model

- Discussed renaming database tables using Meta.db_table

- Clarified Django's project vs app structure
  - Project = PGN Lens (the whole thing)
  - Apps = modules/features within the project (e.g. games, analysis, accounts)
  - Apps are just folders for organizing related functionality
